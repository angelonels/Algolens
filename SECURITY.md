# Security Policy

## Supported Versions

We actively support and fix security issues in the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |
| < 1.0   | :x:                |

---

## Reporting a Vulnerability

We take the security of AlgoLens seriously. If you believe you have found a security vulnerability, please do not open a public issue. Instead, report it privately to our development team.

Please follow this process:

1. **Email the report**: Send an email to [your.email@example.com].
2. **Details to include**:
   - A detailed description of the vulnerability.
   - The potential impact (e.g. what could an attacker gain or do).
   - Clear steps to reproduce the issue (including any code or sample configurations).
   - Any suggestions on how to mitigate or resolve the vulnerability.
3. **Response timeline**: You will receive an acknowledgment of your report within 48 hours. We will keep you updated on our progress as we investigate and work on a fix.
4. **Public disclosure**: We ask that you give us reasonable time to release a patch or mitigation before publicly disclosing the vulnerability.
